Phase 10 Debounced Search + CSV Export ~30 min ⌄
useDebounce Hook
export function useDebounce<T>(value: T, delay = 400): T { const [debouncedValue, setDebouncedValue] = useState<T>(value); useEffect(() => { const timer = setTimeout(() => setDebouncedValue(value), delay); return () => clearTimeout(timer); }, [value, delay]); return debouncedValue; }

Plan Review

What to Fix + What You Got Right

What Your Original Plan Gets Right ✅
  • ✓ Separation of controllers / services / models — evaluators look for this pattern
  • ✓ Zod for validation (not express-validator) — more TypeScript-native choice
  • ✓ TanStack Query v5 — shows you're not using useEffect for data fetching
  • ✓ Custom hooks for debounce and pagination — exactly what they're evaluating
  • ✓ AuthContext pattern — correct global state approach
Recommended Changes ⚠️
  • !Make the services layer real. Controllers must only call service functions — never touch Mongoose directly. If your controller has Lead.find() in it, that's wrong. The service function does the query.
  • !Filter state in URL params. Your plan uses React component state for filters. Use useSearchParams — filters survive refresh and the URL is shareable. Evaluators notice this.
  • !Glassmorphism needs a background. Glass effects on plain #111 look flat. Add a subtle radial gradient or SVG pattern behind the content, then glass cards will actually pop. Use it selectively, not everywhere.
  • !Add createdBy to Lead model. Essential for RBAC ownership check in Phase 5. Missing from your original spec.
  • !Move CSV to a service. Not a separate utility — it belongs in leadsService.ts as an exportLeadsCSV(filter) function called by the export controller.
Automatic Rejection Risks ❌
  • ✗ No .env.example file — instant rejection per brief
  • ✗ Hardcoded localhost:5000 anywhere in frontend — instant rejection
  • ✗ Using any type without justification comment — TypeScript score tanks
  • ✗ Missing API response interfaces in frontend types/api.ts
  • ✗ Comments like // premium color tokens — signals template code to evaluators
  • ✗ console.log statements left in production build
